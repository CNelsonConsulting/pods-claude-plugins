# write-spec

Draft a product requirements document for a proposed feature.

## Inputs

- Problem statement
- Target user/persona
- Success metrics
- Constraints

## Output format

1. Problem and context
2. Goals and non-goals
3. User stories
4. Functional requirements
5. UX and edge cases
6. Metrics and instrumentation
7. Risks and open questions
